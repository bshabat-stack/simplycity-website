# SimplyCity Website

Marketing website for SimplyCity, a local SEO agency for small businesses. Built from a Claude Design export (self-contained HTML pages that boot a small client-side rendering runtime — no build step required).

## Pages

- `index.html` — Home
- `Results.dc.html` — Results / case studies
- `Pricing.dc.html` — Pricing
- `Audit.dc.html` — Free-audit lead capture
- `About.dc.html` — About
- `Contact.dc.html` — Contact
- `Blog.dc.html` — Blog index (`/blog`)
- `BlogPost.dc.html` — Blog post template (`/blog/<slug>`)

Clean URLs are mapped in `vercel.json` (`/about`, `/contact`, `/blog`, …).

## Blog posts

Each locale has its own independent set of posts. They are not translations of
each other and never cross over:

| Folder | URL | Index |
|---|---|---|
| `content/blog/en/` | `/blog/<slug>` | `/blog` |
| `content/blog/es/` | `/es/blog/<slug>` | `/es/blog` |
| `content/blog/he/` | `/he/blog/<slug>` | `/he/blog` |

Posts are Markdown files with frontmatter (`title`, `date`, `excerpt`); the
filename is the URL slug, so `content/blog/en/why-reviews-matter.md` is served
at `/blog/why-reviews-matter` — and at `/es/blog/why-reviews-matter` it returns
a real 404, because no such Spanish post exists. Publishing is just adding one
file to the right folder; see `content/blog/_TEMPLATE.md`. Files starting with
`_` are ignored, as is anything left directly in `content/blog/`.

Each index lists only its own locale's posts, via `api/blog-posts.js?locale=…`;
with zero posts it shows an empty state. `api/blog-post.js` serves the post page
itself and returns a genuine 404 for a slug with no file behind it — a Vercel
rewrite always answers 200, which is why the route goes through a function.
Posts get a self-referencing canonical and no hreflang (they are not
translations of one another); the ten site pages, which are, keep their
clusters. `/sitemap.xml` is generated by `api/sitemap.js` and picks up new posts
with no human step.

## Local preview

Serve the folder with any static file server, e.g.:

```
npx serve .
```

Then open `http://localhost:3000`.

## Deployment

Deployed on [Vercel](https://vercel.com) as a static site — no build command or output directory needed.
