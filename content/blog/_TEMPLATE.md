Files in this folder whose names start with an underscore are ignored by the
blog, so this template never shows up on the site.

To publish a post: copy this file, name it after the URL you want
(`why-reviews-matter.md` becomes `/blog/why-reviews-matter` — lowercase letters,
numbers and hyphens only), fill in the frontmatter above, and write the article
below it in Markdown. That's the whole process — no code changes needed.

`title` is required. A post with no `title` is treated as unpublished: it stays
off the blog index, its URL returns 404, and it is left out of the sitemap.

Nothing else needs editing — in particular, do NOT hand-add the post to a
sitemap. `/sitemap.xml` is generated by `api/sitemap.js`, which reads this
folder on every request and emits the post at all three locales with a full
hreflang cluster. `api/blog-post.js` does the same for the page itself: it
serves the post with a canonical and hreflang tags, and returns a real HTTP 404
for any slug with no file here.

The post body is shared across locales — `/blog/x`, `/es/blog/x` and
`/he/blog/x` all render this one Markdown file, with translated page furniture
around it.
